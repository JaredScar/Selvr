Spatial buckets: nested lists vs count + scatter

Each frame, assign 24 000 point ids to one of 80×45 buckets using the same LCG stream on both sides. TypeScript uses Array<Array<number>> and push per point (many small backing arrays). Selvr-style runs a count pass, builds exclusive prefix offsets, then scatters indices into one Int32Array — the pattern used in GPU radix-style passes and compact sim storage.

Why this is fair

  • Same RNG sequence and bucket formula; both sides clear and refill assignments each frame.
  • Selvr does two passes (count + scatter); TypeScript does one pass with dynamic array growth — comparable total work, different memory behavior.
