API reference

Reference for the selvr command-line tool, shipped std/* modules (from the repository source), compiler attributes, embedded JS runtime helpers, and linter codes. For language syntax and semantics, see the Selvr specification.

Overview

Selvr compiles .self sources to JavaScript (and optional hybrid WASM artefacts). Standard library APIs are defined in std/ in the repo; examples and the spec sometimes use logical paths such as std/web/dom — map those to the on-disk modules listed under Import paths.

This page is static HTML in docs/api/ for GitHub Pages. Regenerate or extend it when you add exports to std/*.self or new CLI subcommands.

Minimal program

fn main(): void {
  console.log("Hello, Selvr");
}

selvr CLI

Subcommands exposed by the compiler binary (crates/selvr-cli):

CommandDescription
selvr build <file.self>Compile to JS (default) or hybrid output via --emit js|hybrid. Optional -o / --output, --source-map.
selvr run <file.self>Compile and execute with Node.js.
selvr check <file.self>Typecheck / parse without emitting output.
selvr dump <file.self>Print AST or IR (--format ast|ir).
selvr explain <file.self>Print per-function WASM vs JS targeting decisions; --json for machine-readable output.
selvr add / install / removePackage manifest (selvr.toml) and lockfile maintenance.
selvr publishPublish package to the registry (SELVR_TOKEN).
selvr search <query>Search the package registry.
selvr test <file|dir>Run #[test] functions.
selvr fmt [files…]Format sources; --check for CI.
selvr init [name]Scaffold selvr.toml and entry file.
selvr lint [files…]Lint (--json, --strict).
selvr lspLanguage Server Protocol (stdio).
selvr dapDebug Adapter Protocol (stdio).

Shell

# Emit JavaScript (default)
selvr build src/app.self -o dist/app.js

# Hybrid WASM + JS bridge + loader
selvr build src/app.self --emit hybrid -o dist/app

# Typecheck only, CI-friendly format / lint
selvr check src/app.self
selvr fmt src/ --check
selvr lint src/ --strict

# Targeting report (per function)
selvr explain src/app.self
selvr explain src/app.self --json

# Run tests and execute with Node
selvr test tests/
selvr run examples/01_hello_world.self

Compiler attributes

Targeting and FFI hints (see spec and selvr explain):

AttributeMeaning
#[js]Force emission / scoring toward the JavaScript runtime (DOM, timers, fetch).
#[wasm]Force WASM-side placement for compute-heavy code (when using hybrid emit).

Selvr

// Pin hot UI glue to JS and numeric kernels to WASM (hybrid emit).
#[js]
export fn mount(root: Element): void {
  root.addEventListener("click", (_) => console.log("click"));
}

#[wasm]
export fn dot_product(a: f64[], b: f64[], n: i32): f64 {
  let s: f64 = 0.0;
  let i: i32 = 0;
  while i < n {
    s += a[i] * b[i];
    i += 1;
  }
  return s;
}

Embedded JS runtime (__selvr)

Phase-1 JS output prepends a small __selvr object for Option, Result, unwrap, and panic. Range-based for loops may lower to __selvr.range in some builds — ensure runtime helpers match your compiler version.

Generated JS (excerpt)

// After compile, Option/Result helpers are plain objects:
const name = __selvr.unwrap(maybe);  // throws if None
if (__selvr.isSome(opt)) { /* ... */ }

std/dom/dom.self — DOM & window

Element selection, mutation, events, and helpers. Aliased in examples as std/web/dom.

Direct browser APIs (single-file friendly)

fn main(): void {
  const el = document.getElementById("out");
  if el {
    el.textContent = "Hello";
    el.addEventListener("click", () => console.log("ok"));
  }
}

Std wrappers (idiomatic)

import { by_id, set_text, create, append_child, on } from "std/web/dom";

fn main(): void {
  const root = by_id("app");
  if root {
    const p = create("p");
    set_text(p, "Hello from std/dom");
    append_child(root, p);
    on(p, "click", (_) => set_text(p, "Clicked"));
  }
}

std/net/fetch.self — HTTP

Async fetch helpers. Import in examples as std/web where re-exported.

Selvr

import { get, get_json, post_json, is_ok_status } from "std/net/fetch";

async fn main(): void {
  const html = await get("https://example.com/");
  console.log(html.length);

  const doc = await get_json("https://api.github.com/repos/JaredScar/Selvr");

std/net/websocket.self

import {
  ws_connect, on_open, on_message, send_text, ws_close, is_open,
} from "std/net/websocket";

fn main(): void {
  const ws = ws_connect("wss://echo.websocket.events");
  on_open(ws, (_) => send_text(ws, "hello"));
  on_message(ws, (ev) => console.log(ev.data));
  if is_open(ws) {
    ws_close(ws);
  }
}

std/collections/array.self

Numeric / buffer-oriented helpers (mostly i32 / f64).

Selvr

import { sort_i32, dedup_i32, slice_i32 } from "std/collections/array";
import { sum_i32 } from "std/core/iter";

fn stats(): i32 {
  let v: i32[] = [9, 3, 7, 1, 3];
  sort_i32(v, v.length);
  const uniq = dedup_i32(v, v.length);
  const head = slice_i32(uniq, 0, 3);
  return sum_i32(head);
}

std/core/option.self

Helpers typed around Option<i32> in the current std (narrower than generic Option<T> in examples).

Selvr

import { unwrap_or, is_none } from "std/core/option";
import { i32_to_string } from "std/collections/string";

fn display_id(m: Option<i32>): string {
  if is_none(m) {
    return "none";
  }
  return i32_to_string(unwrap_or(m, 0));
}

Import paths (spec vs repository)

The specification lists logical web modules. Map them to files under std/ in this repo:

Typical import block

// Logical paths from SPEC / examples map to these on-disk modules:
import { query, create, on } from "std/web/dom";       → std/dom/dom.self
import { get, post_json } from "std/net/fetch";
import { ws_connect, on_message } from "std/net/websocket";
import { trim, split } from "std/collections/string";
import { sum_i32, range } from "std/core/iter";
import { unwrap_or } from "std/core/option";
Documented / example importRepository module
std/web/domstd/dom/dom.self
std/web (fetch)std/net/fetch.self
std/web/socketstd/net/websocket.self
std/web/canvas, std/web/time, std/web/eventsSee examples / future std layout; not all are split as separate files yet.

When in doubt, search export fn under std/ in the repository.
