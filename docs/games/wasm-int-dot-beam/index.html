Integer dot product: WASM vs JavaScript

500 rounds per frame: randomize two length-8 192 i32 vectors in linear memory, then compute Σ a[i]*b[i] with 32-bit wrapping. The WASM side runs the inner loop as a native i32 kernel; TypeScript runs the same arithmetic as JavaScript number semantics on the right — illustrating how Selvr can keep integer-heavy work in WASM while TS stays on the JS tier by default.

Selvr vs TypeScript

For fixed-width integer pipelines, WASM matches machine semantics directly. TypeScript always targets JavaScript’s number model on this path unless you introduce another compiler. Selvr’s hybrid split is meant to automate “integer/float kernel → WASM, glue → JS” so you are not forced to choose between ergonomics and a second toolchain for every hot loop.
