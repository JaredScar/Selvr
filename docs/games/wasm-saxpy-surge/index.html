Level-1 BLAS: SAXPY in WASM vs JavaScript

Each frame runs 100 rounds of y += a·x on length-4 096 vectors (randomized each round). The left panel runs the update loop in WebAssembly; the right uses TypeScript→JavaScript. This is the same split Selvr advertises for linear-algebra kernels: WASM for the heavy multiply-add sweep, JS for setup and rendering.

Selvr vs TypeScript

TypeScript does not give you WASM for for-loops automatically; you stay in JS unless you add a second toolchain. Selvr’s hybrid compiler can emit this style of kernel to WASM without you maintaining a separate C/Rust pipeline.

Rebuild TypeScript bundle

npx esbuild game.ts --bundle --format=esm --outfile=game.js from typescript/
