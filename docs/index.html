Performance demos (particles, grids, batch math), correctness, and skill — hub with full list.

Dot Wave

WASM dot kernel + JS canvas vs TypeScript (JavaScript-only) — same float workload.

vs TypeScript: hybrid WASM+JS mirrors Selvr’s split; TS compiles to JS unless you bolt on WASM manually.

Fib Rush

25k× fib(35) per frame: WASM integer loop vs JavaScript.

vs TypeScript: same algorithm, different execution tier — compare compute ms on the two panels.

Particle Storm

48k particles: struct-of-arrays (Selvr-style) vs heap objects — FPS & physics time.

vs TypeScript: packed lanes vs Particle[] — fewer references and better cache use on the hot integration loop.

Grid Life

Conway 180×200: flat Uint8Array vs boolean[][] — cache-friendly grid kernels.

vs TypeScript: one buffer vs many row arrays — neighbor stencil reads become stride-friendly.

Vector Bench

30k normalizes: packed Float64Array vs Vec3[] — batch linear algebra.

vs TypeScript: AoS scatters loads across objects; SoA keeps components contiguous for the normalize kernel.

Alloc Strike

Per-frame Float64Array allocation vs reused buffer — allocator & GC pressure.

vs TypeScript: reuse avoids malloc + GC every frame — steadier frame times.

Path BFS

BFS flood fill: shift() queue vs ring buffer; nested visited vs Uint8Array.

vs TypeScript: ring dequeue is O(1); shift() rewrites the whole queue each step.

Count Sort Surge

100k-key counting sort: number[] vs Uint32Array pipeline.

vs TypeScript: typed integers avoid boxed elements in tight counting loops.

Matrix Core

Dense 32×32 matmul: number[][] vs row-major Float64Array.

vs TypeScript: row-major flat storage removes per-row pointer hops in the inner multiply.

Bucket Downpour

Spatial buckets: nested arrays + push vs count + Int32Array scatter.

vs TypeScript: count + scatter into one slab cuts growth churn from many small arrays.

Queue Blitz

Array.shift FIFO vs O(1) ring buffer on fixed memory.

vs TypeScript: worst-case stress test for shift — ring stays constant-time.

Spring Chain

3.8k spring masses: Float64Array lanes vs { x, v }[] — integration & draw.

vs TypeScript: SoA removes a heap object per node; forces use sequential index math on dense buffers.

Blur Pass

120×120 box blur: Uint8Array planes vs number[][] stencil.

vs TypeScript: flat bytes vs nested rows — less indirection and boxing on every neighbor sample.

Binary Hunt

20k binary searches/frame on 65k keys: Uint32Array vs number[].

vs TypeScript: same algorithm; fixed-width slab improves locality vs boxed array slots.

Core Harvest

Same +1 harvest: TypeScript number hits IEEE-754 limits; Selvr uses exact integers.
