Grid blur: nested arrays vs one buffer

Each frame applies a 3×3 box blur on a 120×120 byte grid (edge pixels unchanged). TypeScript uses number[][] — each row is a separate array, so every sample is a double indirection plus boxed numbers. Selvr-style uses two swapped Uint8Array planes: dense memory, predictable stride, no per-cell heap object.

Speed vs TypeScript

Expect lower compute ms/frame on the packed side: stencil kernels are memory-bound; a flat typed array keeps the working set in cache-friendly order, while number[][] scatters loads across row objects and often boxes values as IEEE-754 heap numbers.
