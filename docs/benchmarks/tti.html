Selvr Performance

Cold Start · Time-to-Interactive · Runtime FPS Phase 2.4

1 — Cold Start & Parse Time measured

Time for the browser's JS engine to parse a bundle of source code using new Function(src). Selvr compiles to a minimal, tree-shaken JS bundle — no framework overhead, no dead code. Measured by forcing a full re-parse in a tight loop and averaging.

2 — Time-to-Interactive measured

TTI measures how long it takes from page load until an interactive element responds to user input. "Selvr" here is the compiled JS output (as produced by selvr build), measured by immediately invoking a simulated setup function after parse and recording the elapsed time. React/Angular TTI includes their framework bootstrap overhead.

3 — Runtime Animation FPS measured

Sustained frame rate under computation load. A particle simulation runs: once in vanilla JS (tight loop, no GC pause suppression) and once in Selvr-compiled JS (same algorithm, as produced by the Selvr compiler — already optimised by the targeting pass). The WASM path is the Phase 2 target; JS results are shown here.
