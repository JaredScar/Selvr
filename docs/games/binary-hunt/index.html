Many binary searches: array layout

A sorted table of 65 536 integers; each frame runs 20 000 lower_bound-style binary searches on random keys. TypeScript keeps the table in a number[] (boxed doubles, pointer chasing). Selvr-style uses a Uint32Array — fixed-width elements in one slab, friendlier to the CPU cache on random-ish probes.

Speed vs TypeScript

The algorithm is identical; the difference is memory representation. Expect modestly lower compute ms with Uint32Array when the array stops fitting cleanly in cache — fewer loads from main memory and no per-element boxing overhead on the hot path.
