A new language for the web

The web deserves a better language than JavaScript.

Selvr is a statically-typed, compiled language that runs natively in the browser. Familiar syntax for TypeScript developers. Sound types. No null. No GC pauses. Dramatically faster cold starts.

The Problem
JavaScript was never designed for this
JavaScript was created in 10 days in 1995 to add small scripts to web pages. Today it powers billion-dollar applications, machine learning models, and real-time games — on the same foundation. The cracks are showing.
🕳️
The Billion-Dollar Mistake
JavaScript inherited null and undefined. TypeScript adds types on top, but can't remove them. Every ?.something is a runtime surprise waiting to happen.
// TypeScript — still compiles: const x: string = null as any; x.toUpperCase(); // 💥 TypeError at runtime
🤫
Silent Errors
Functions that throw in TypeScript look identical to functions that don't. There's no way to know a function can fail without reading its implementation or documentation.
// No indication this can throw: async function loadUser(id: number) { return await fetch(`/api/${id}`); // throws! }
🐌
The Parse Tax
Every cold page load pays a tax: the browser must download, parse, and JIT-compile your JavaScript bundle before showing anything. A 300 KB React app adds 150–400 ms on mobile.
// 300 KB React app on a mid-range phone: // ~180ms just to parse main.chunk.js // Before a single pixel is rendered.
🗑️
GC Pauses
JavaScript's garbage collector can pause your program for 10–50 ms at any moment. For animations running at 16 ms per frame, a single GC pause causes a visible stutter that destroys the experience.
// A GC pause of 32ms during an animation: // Drops from 60fps → 30fps for one frame // User sees a "jank" — and blames your app.
The Solution
Selvr: designed for the web of today
Selvr is not a superset of JavaScript. It's a new language that keeps everything developers love about TypeScript's syntax, while fixing the fundamental design mistakes at the root.
01 — Type Safety
No any. No null.
If it compiles, it's type-safe. Period. There is no escape hatch. any does not exist. null and undefined do not exist. Absence is modelled with Option<T> — and the compiler enforces every access.
02 — Error Handling
Errors are values.
Functions that can fail return Result<T, E>. You can see from any signature exactly what can go wrong. The compiler prevents you from ignoring errors. The ? operator makes propagation one character.
03 — Performance
No GC. No parse tax.
Selvr compiles to bytecode — no JS parser at runtime. The ownership model frees memory deterministically at compile time. No garbage collector means no GC pauses, ever. Smooth animations, fast cold starts.
04 — Familiar Syntax
Reads like TypeScript.
Colons for return types. Arrow functions. Template literals. const and let. async/await. If you know TypeScript, you can read Selvr immediately. The learning curve is adding Option, Result, and match.
05 — Web Native
DOM & fetch are stdlib.
The standard library includes typed bindings for the DOM, fetch, WebSockets, Canvas, timers, and events. There's no FFI, no @types/ packages to install, no any escape for browser APIs.
06 — Exhaustive Matching
match that works.
Selvr's match expression is exhaustive by default. Adding a new variant to an enum turns every non-exhaustive match into a compile error. TypeScript's switch requires a never-trick for the same guarantee.

Target Audience
Who should use Selvr?
Selvr is designed for developers who care about correctness and performance — and are willing to spend a few hours learning a better set of tools.
🟦
TypeScript developers
The closest audience. You already understand types, generics, and async/await. Selvr adds what TypeScript can't: sound types, no null, and deterministic memory.
  • Syntax is 90% identical
  • No null — already fighting this
  • Result replaces try/catch
  • match replaces switch
🦀
Rust developers building for the web
You know ownership and Result. Selvr brings those concepts to the browser with a JS-adjacent syntax — no wasm-bindgen, no manual FFI.
  • Familiar ownership model
  • Option and Result as first class
  • Compiles to browser-native bytecode
  • DOM stdlib — no glue code
🎮
Game & animation developers
GC pauses are your worst enemy at 60fps. Selvr's ownership model eliminates them. The animation loop runs at consistent frame times — no jank from the garbage collector.
  • Zero GC pauses
  • Predictable frame times
  • Canvas API in stdlib
  • requestAnimationFrame typed
Scope
What Selvr is NOT
Being clear about non-goals is as important as defining goals. Selvr will not try to be everything.

Explicit non-goals

  • A JavaScript superset — Selvr is a clean-slate language, not TypeScript++
  • General-purpose systems programming — use Rust for that
  • A server-side language — web-first; server target is a Phase 3 stretch goal
  • Backward compatible with existing JS/TS codebases
  • A replacement for Rust or C++ for embedded systems
  • Zero learning curve — some investment is required and expected
